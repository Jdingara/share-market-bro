Personal Project · Built July 2026

Share Market Bro

An autonomous bot that watches the Indian stock market, decides for itself when to trade NIFTY 50 options, and manages its own risk — tested thoroughly with fake money before it ever touches real capital.

1 What is this project?

A computer program that trades on your behalf — but only when it finds a genuinely good opportunity, and never more than once a day.

Every trading day, from market open to close, the bot continuously reads live price data for the NIFTY 50 index and analyzes it using a mix of statistical rules and machine learning. When — and only when — it finds a setup it's genuinely confident about, it "buys" an option (a call if it expects prices to rise, a put if it expects them to fall), then automatically exits at a fixed +10% profit target or a -5% loss limit, whichever comes first.

Right now, everything runs on simulated ("paper") money — real market prices, real decisions, zero real financial risk — so the strategy can be proven before it's trusted with actual capital.

2 What does the bot actually do, step by step?

The same disciplined loop, every single trading day.

9:15 AM

Logs into the trading account automatically — no manual steps.

All day

Watches live prices, checking every minute for a high-confidence setup.

If found

Picks the right option (call or put), enters at the real market price.

Then

Watches that position closely, exits automatically at +10% or -5%.

Done for the day

Win or lose, it stops. No second trade, no chasing losses.

If nothing found

It does nothing. A quiet day is a normal, expected outcome — not a failure.

3 What market does it trade?

NIFTY 50 Index Options — not individual company shares. NIFTY 50 tracks India's top 50 listed companies as a single index, and "options" are contracts that let you bet on its direction (up via a Call, down via a Put) without owning the index itself.

Broker: Zerodha (Kite Connect API) Instrument: NIFTY 50 weekly options Lot size: 65 Min. 3+ days to expiry

5 The full technical stack, explained simply

Every app has three basic pieces: something that thinks (backend), something that runs it (server), and something you look at (frontend). Here's what each one is in this project specifically, and every tool used, explained in plain terms.

🧠 Backend

The "thinking" part — nobody sees it directly

All written in Python. This is every piece of logic: connecting to the broker, analyzing prices, deciding buy/sell, running the ML models, tracking money. About 20 files, each with one clear job (see Section 4's phases).

🖥️ Server

Where it actually runs

There is no cloud server — everything runs on this one personal computer. Streamlit creates a tiny local web server on this machine (address localhost:8501), which only this computer can open in a browser. Nothing is exposed to the internet.

🎨 Frontend

What you actually see and click

The dashboard webpage — Start/Stop buttons, live activity log, win-rate and profit numbers, trade history tables and charts. Built with Streamlit, which turns plain Python code directly into a webpage, no separate web-design work needed.

6 The "brain" — models & technology used

Two different approaches to deciding when to trade, built and compared honestly against each other.

Approach A — Rule-Based Analysis

Classic technical analysis, combined together:

  • EMA (Moving Averages) — reads the overall trend direction
  • Fibonacci Retracement — finds key support/resistance price levels
  • RSI (Momentum) — confirms real strength behind a move
  • Candlestick Patterns — confirms a reversal is actually happening
  • Bollinger Bands — flags when price is at a statistical extreme

Approach B — Machine Learning

Models that learn patterns instead of following fixed rules — three were built and honestly compared:

  • Logistic Regression — the simplest model, a straight dividing line through the data
  • Random Forest — an ensemble of decision trees
  • Gradient Boosting (XGBoost) — trees that each correct the previous one's mistakes
  • 16 engineered features per moment in time (trend strength, momentum, volatility, time of day, etc.)
  • Chronological validation — tested only on data never seen during training, to keep results honest
Winner: Gradient Boosting (XGBoost) — beat both other models on every measure (see Results below) and is now the model actually running live. Logistic Regression coming out weakest was actually a useful finding: it proved the real pattern in the data is not a simple straight line, which is exactly why the more sophisticated tree-based models do better — not complexity for its own sake.
Options Pricing Model: Since real historical option prices aren't available from the broker, the bot uses the Black-Scholes model — the industry-standard mathematical formula for pricing options — to realistically simulate what an option's price would have done during backtesting.

It still came out clearly ahead overall (roughly double the P&L of the old -10% stop, on the same real trades) because cutting the size of confirmed losses mattered more than the few early exits cost - but it's a genuine trade-off, not a guaranteed win in every scenario.

9 Safety & discipline built in

  • Maximum one trade per day — no exceptions, no chasing losses
  • Automatic +10% profit target and -5% stop-loss on every trade (tightened from -10% on 2026-07-10, after real evidence showed it performs better)
  • Will do nothing at all on days with no genuinely good setup — never forces a trade
  • Fully tested on 132 days of real historical data before going live with even fake money
  • Currently running on simulated money only — real prices, zero real financial risk
  • Real position sizing — only buys as many whole option lots as the current paper balance can actually afford (never a fractional or forced position); sits out entirely if funds are too low
  • Hard cap: never more than ₹2,00,000 risked on any single trade — even if the account compounds to a much larger balance over time, one trade can never risk more than this fixed ceiling; any balance above the cap simply sits idle rather than being deployed
  • A visual dashboard (web page) to start/stop it and see every trade, win rate, and profit/loss at a glance
  • Automatically recovers from network hiccups instead of silently crashing during the trading day
  • Source code kept in a private GitHub repository — credentials and trade history never included
